# ================================================ #
#       正交坐标系／ortho 注意事项
# ================================================ #

正交坐标系 的裁截空间是个 立方体盒子，需要定义其 六个面的边界

#  正确的写法如下：
glm::ortho(-100.0f,  -- 左边界 
            100.0f,  -- 右边界
           -100.0f,  -- 上边界
            100.0f,  -- 下边界
           -100.f,   -- 近平面
           1000.0f   -- 远平面
           );

#  错误的写法如下：
glm::ortho( 0.0f,   //-- 左边界
            800.0f, //-- 右边界
            0.0f,   //-- 上边界
            600.0f, //-- 下边界
            -10.1f,   //-- 近平面
            100.0f  //-- 远平面
            );

#  由于 物体在世界坐标系中，常以 0 点为中心。
#  在错误写法中，0点被 放到了 window 左下角。自然容易找不到。


# --------------------------------- #
#    将世界坐标系的 1 和 1个像素 对应
# --------------------------------- #
为了实现这一点，需要做以下调整。
假设，
游戏的 显示 window 为 800 * 600 (像素)
游戏的 运算 window 为 120 * 90 (像素)
游戏中某个图元，尺寸为 16 * 16 (像素)

# -1-
首先，这个图元的 物体空间坐标应该为：
float rectangle[] = {
    //---   坐标            textuel  
    -8.0f, -8.0f, 0.0f,   0.0f, 0.0f,
     8.0f, -8.0f, 0.0f,   1.0f, 0.0f,
     8.0f,  8.0f, 0.0f,   1.0f, 1.0f,
     8.0f,  8.0f, 0.0f,   1.0f, 1.0f,
    -8.0f,  8.0f, 0.0f,   0.0f, 1.0f,
    -8.0f, -8.0f, 0.0f,   0.0f, 0.0f,
};

可以看到，坐标值和像素值 是完全对应的。

# -2-
然后，设置 camera 正交窗口为
glm::ortho(-60.0f,  -- 左边界 
            60.0f,  -- 右边界
           -45.0f,  -- 上边界
            45.0f,  -- 下边界
           -100.f,   -- 近平面，尽可能地近
           1000.0f   -- 远平面，尽可能地远
           );

可以看到，坐标值和 运算window 的像素值 ( 120 * 90 ) 也是完全对应的。
